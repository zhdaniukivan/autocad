

### 1. Классификация блоков

| Тип шкафчика                  | isBottom | isTop | Учитывается в нижней длине | Учитывается в верхней длине | Примечание |
|-------------------------------|----------|-------|----------------------------|-----------------------------|----------|
| **Нижний ряд**                | 1        | 0     | Да                         | Нет                         | Только низ |
| **Верхний ряд**               | 0        | 1     | Нет                        | Да                          | Только верх |
| **От пола до потолка**        | 1        | 1     | Да                         | Да                          | Участвует в обеих линиях |
| **HeightsForCabs**            | —        | —     | —                          | —                           | Специальный блок высот |

---

### 2. Пересчёт ширины (ChangeWidth)

**Нижняя линия:**
- Сумма `Width` всех блоков, где `isBottom = 1` (включая full-height)

**Верхняя линия:**
- Сумма `Width` всех блоков, где `isTop = 1` (включая full-height)

**Логика распределения:**
- При задании новой целевой ширины для нижнего или верхнего ряда рассчитывается разница (Δ).
- Δ делится **только** на количество блоков с `ChangeWidth = 1` в этом ряду.
- Каждый такой блок изменяется на `Δ / количество`.

---

### 3. Специальные элементы (планки и столешница)

| Элемент              | Свойства для изменения          | Какая сумма используется                          | Примечание |
|----------------------|----------------------------------|----------------------------------------------------|----------|
| **Верхняя планка**   | `upper_filler_w`, `Recalculated_Width` | Только **чистые верхние** (`isTop=1` **и** `isBottom=0`) | **НЕ** включает full-height |
| **Столешница**       | `countertop_w`, `Recalculated_Width` | Только **чистые нижние** (`isBottom=1` **и** `isTop=0`) | — |
| **Нижняя планка**    | `kick_plate_w`, `Recalculated_Width` | Только **чистые нижние** (`isBottom=1` **и** `isTop=0`) | — |

---

### 4. Логика высоты кухни (HeightsForCabs)

Блок: **`HeightsForCabs`** (EffectiveName)

Содержит свойства:
- `Bottom_Height` (обычно 915)
- `Middle_Height` (610)
- `Upper_Height` (850) ← **это поле нужно обновлять**
- `UpperFiller_Height` (50)
- `Floor_Height` (13)
- `Opening_Height` (2438) — общая высота проёма

**Алгоритм при изменении общей высоты кухни:**
1. Пользователь вводит желаемую **общую высоту** (`Opening_Height`).
2. Вычисляется разница: `Δ = новая_общая_высота - текущая_общая_высота`.
3. Эта разница применяется ко всем шкафчикам, `ChangeHeight = 1`).
4. В блоке `HeightsForCabs` обновляется значение `Upper_Height` = `Upper_Height + Δ`.

---

### 5. Автоматическое обновление спецэлементов (рекомендация)

**Да, рекомендуется обновлять автоматически** после каждого успешного изменения ширины в групповом режиме:

- После изменения нижней линии → обновить `countertop_w`, `kick_plate_w` и их `Recalculated_Width`
- После изменения верхней линии → обновить `upper_filler_w` и `Recalculated_Width`

Это сильно снизит вероятность ошибок у пользователя.

### Итоговая схема зависимостей (коротко):

ЭТУ ПРОВЕРКУ МОЖНО ВЫПОЛНЯТЬ ПРИ ВЫДЕЛЕНИИ БЛОКОВ И ПРОСТО ВЫВОДИТЬ ПРОВЕРКИ ПРОШЛИ УСПЕШНО, ЛИБО ПРОВЕРКА …. ПРОВАЛИСЛАСЬ, БУДЬТЕ ВНИМАТЕЛЬНЫ.

- **Нижняя длина** = `sum(Width | isBottom == 1)`
- **Верхняя длина** = `sum(Width | isTop == 1)`
- **Upper Filler** = `sum(Width | isTop == 1 and isBottom == 0)`
- **Countertop = Kick Plate** = `sum(Width | isBottom == 1 and isTop == 0)`
